Form

정의

사용자 입력을 수집하는 데 사용됩니다. 사용자 입력은 서버로 전송됩니다.

요소

input

input 요소는 form안에서 사용자로부터 입력을 받을 수 있는 input field를 생성합니다.

type

  • 텍스트: 단일 줄 텍스트 입력 필드
  • 라디오 : 다양한 선택 항목 중 하나 선택하기 위한 버튼 표시
  • 체크박스 : 여러 개의 선택 항목 중 하나 이상 선택하기 위한 버튼 표시
  • 제출 : 제출 버튼 표시
  • 버튼 : 클릭 가능한 버튼 표시

name

input 요소의 name 속성은 form data를 서버로 전송할 때 form data object(FormData)의 key로 사용합니다.

label

해당 요소는 여러 개의 양식 요소에 대한 레이블을 정의한다. 사용성에 중요하며 라디오 버튼, 체크박스 등을 클릭하기 위해 요소 내의 텍스트를 클릭하면 된다.

select

select 요소는 다양한 선택 항목 중 하나를 선택할 수 있는 드롭다운 목록을 생성합니다.

option

option 요소는 select 요소 내에서 선택할 수 있는 항목을 정의합니다.

selected

option 요소의 selected 속성을 추가하면 미리 선택된 옵션이 정의 됩니다.

size

select 요소의 size 속성은 드롭다운 목록의 표시할 항목의 수를 결정합니다.

multiple

multiple 속성을 추가하면 select 요소 내에서 여러 개의 항목을 선택할 수 있다.

textarea

textarea 요소는 여러 줄의 텍스트를 입력할 수 있는 textarea field를 생성합니다.

rows

텍스트 영역에 표시되는 줄의 개수

cols

텍스트 영역의 표시 너비

button

클릭 가능한 버튼 정의

fieldset, legend

fieldset 요소는 form 안에서 form data를 묶어서 그룹화할 수 있다. legend 요소는 fieldset 요소의 제목을 정의한다.

datalist

datalist 요소는 input 요소의 value attribute를 auto-complete 기능을 위해 사용할 수 있다.

output

output 요소는 계산의 결과(스크립트로 수행된 결과 등)을 나타낸다.

속성

value

input, textarea, select 요소의 초기값

readonly

input, textarea, select 요소의 value를 수정할 수 없는지 여부

disabled

input, textarea, select 요소를 비활성화

size

input, textarea, select 요소의 표시 너비

maxlength

input, textarea, select 요소의 최대 문자 수

min, max

input, textarea, select 요소의 최소, 최대값

multiple

select 요소의 multiple 속성을 true로 설정하면 여러 개의 option을 선택할 수 있다.

pattern

input, textarea, select 요소의 value attribute가 pattern attribute와 일치하지 않으면 validation error를 발생

placeholder

input, textarea, select 요소의 예상값

required

input, textarea, select 요소의 required 속성을 true로 설정하면 value attribute가 필수로 되어야 validation error를 발생

step

input, textarea, select 요소의 step attribute가 1일 때, value attribute는 min, max attribute로 지정한 min, max value 사이에서 step size만큼 증가/감소할 수 있다.

autofocus

input, textarea, select 요소에 focus를 주어서 input, textarea, select 요소의 value attribute를 쉽게 입력할 수 있도록 한다.

datalist

datalist 요소의 id attribute로 input, textarea, select 요소의 list attribute로 지정한 datalist 요소를 참조할 수 있다.

action

form data를 전송할 URL

target

수신된 응답을 표시할 위치 지정. 새 브라우저 탭에서 열릴지 등을 결정한다

method

form data를 전송할 HTTP method. 폼 데이터는 URL 변수(get) 또는 HTTP 게시 트랜잭션(POST)로 전송 가능하다. GET, POST, PUT, DELETE 중 선택

get

이름/값 쌍으로 URL에 데이터를 추가한다. 민감한 데이터를 보낼 때는 사용 불가하다. 제출된 양식은 URL에서 볼 수 있기 때문이다. URL의 길이는 2048로 제한되며, 사용자가 결과를 북마크하려는 양식 제출, Google의 쿼리 문자열과 같은 비보안 데이터에 적합하다.

post

HTTP 요청 본문 내부에 양식 데이터를 추가한다. URL에 표시 되지 않는다. 크기 제한이 없고, 제출한 form을 북마크할 수 없다.

autocomplete

자동 완성 기능 on/off

Novalidate

양식 데이터의 유효성 검사 하지 않음을 지정

입력 양식 속성

폼 속성

input 속성은 input요소가 속한 form 양식을 지정한다. 이 속성의 값은 해당 속성이 속한 form 요소의 id 속성과 동일해야 한다.

formaction

form data를 전송할 URL. 이 속성은 action 속성보다 우선한다. 이 속성은 제출 및 이미지 입력 유형과 함께 작동한다.

formenctype

form data의 인코딩 방식. 이 속성은 enctype 속성을 재정의한다. 이 속성은 form data를 URL 변수(get) 또는 HTTP 게시 트랜잭션(POST)로 전송할지, form data를 HTTP 쿠키(Cookie)로 전송할지를 결정한다. application/x-www-form-urlencoded, multipart/form-data, text/

formmenthod

form data를 전송할 HTTP method. 이 속성은 form data를 URL 변수(get) 또는 HTTP 게시 트랜잭션(POST)로 전송할지, form data를 HTTP 쿠키(Cookie)로 전송할지를 결정한다. get, post, PUT, DELETE 중 선택

formtarget

수신된 응답을 표시할 위치 지정. 새 브라우저 탭에서 열지 등을 결정한다

formnovalidate

양식 데이터의 유효성 검사 하지 않음을 지정
